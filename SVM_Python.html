def predictAndPlot(x, m):
    #m = SVC().fit(x, y)
    
    ## Predict on grid
    nGrid = 40
    range1 = np.linspace(x[:,0].min(), x[:,0].max(), nGrid)
    range2 = np.linspace(x[:,1].min(), x[:,1].max(), nGrid)
    (x02d, x12d) = np.meshgrid(range1, range2)
    x0 = x02d.ravel()
    x1 = x12d.ravel()
    xp = np.column_stack((x0, x1))
    # xp is a new X matrix using the points on regular grid.
    # now predict the results using that matrix
    yp = m.predict(xp)
    ypNN2d = yp.reshape((40,40))
    
    ## plot the graph
    plt.imshow(ypNN2d, alpha=0.3, extent=[x0.min(), x0.max(), x1.min(), x1.max()], origin='lower', aspect="auto")

    plt.contour(x02d, x12d, ypNN2d, levels=[0.5], colors='black')
    plt.contour(x02d, x12d, ypNN2d, levels=[1.5], colors='black')
    # also add the observed values as dots here
    plt.scatter(x[:,0], x[:,1], c=y, alpha=0.5, s=100)

Comparing the performances of the different types of kernels, we can see that the sigmoid kernel performs the worst. This might be due to the reason that sigmoid function returns two values, 0 and 1,and hence it is more suitable for binary classification problems. As we have three output classes, sigmoid doesnt suit well.

The other two kernels, polynomial and radial performed better but with few mis-classifications.
